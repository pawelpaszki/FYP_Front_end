The main purpose of this application is to display image freshness of a number of sample Docker images. Those images are analyzed in daily automated tests to find and record vulnerable components in the source code. As a non-registered user, you can see the image freshness of the images mentioned above in Images tab. Additionally, a graphical representation of the number of vulnerable components based on their severity (low, medium or high) is possible to generate over a selected range of time, when clicking icon chart button.

The image freshness definition is displayed, when hovering over an "i" icon beside "Freshness grade" heading.

To generate the vulnerability chart, from and to dates have to be selected from the calendar.

The generated chart will show the details of vulnerable components based on their severity per selected time range. Below the chart, the list of all recorded vulnerable components over each of the days in range can also be investigated by pressing the vulnerability details button.

Registered and authenticated users are allowed to access several additional features. Such users can pull a new image from Docker Hub from Search Docker Hub tab.

An authenticated users can also remove images without image freshness established.

From Containers tab authenticated users can also perform number of checks of the source code of a selected image.

After successful execution of those checks, their results will be available to see.

From Containers tab individual containers can be stopped, started and removed, depending on their current state.

From Account tab it is possible to update the password
